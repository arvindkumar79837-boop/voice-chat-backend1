## Vulnerability Description

### Before Fix
- **App sent raw provider data** (`providerUid`, `email`, `displayName`, `photoUrl`) from the client
- **Backend trusted these values blindly** — anyone could craft a request with any `providerUid` and `email`
- **Result:** Complete account takeover — attacker could impersonate any Google/Apple user by supplying their UID

### After Fix
- **App sends Firebase ID Token** (`idToken`) — signed by Firebase, cryptographically verified
- **Backend verifies token server-side** using `admin.auth().verifyIdToken(idToken)`
- **Extracted claims are used** — `uid`, `email`, `name`, `picture` come from verified Firebase token
- **Provider confirmed** — `decodedToken.firebase.sign_in_provider` checked against claimed provider

## Changes Made

### Backend (`repo2`)

**`src/controllers/authSecure.controller.js`:**

1. **Added `firebase-admin` import** — `const admin = require('firebase-admin')`
2. **Rewrote `socialLogin`** (line 328):
   - Accepts `{ provider, idToken, deviceInfo }` — removed `providerUid`, `email`, `displayName`, `photoUrl` params
   - Calls `admin.auth().verifyIdToken(idToken)` to verify token server-side
   - Extracts verified `uid`, `email`, `name`, `picture` from decoded token
   - Confirms `decodedToken.firebase.sign_in_provider` matches claimed provider
   - Returns 401 if token invalid/expired, 400 if provider mismatch
3. **Rewrote `linkSocialAccount`** (line 420):
   - Same pattern — accepts `{ provider, idToken }`, verifies server-side
   - No longer trusts client-supplied data

### App (`repo1`)

**`lib/features/auth/presentation/repositories/auth_repository.dart`:**
- `socialLogin()` signature changed from `(provider, providerUid, email, displayName, photoUrl)` to `(provider, idToken)`

## Remaining Concerns

1. **Admin SDK initialization** — `admin.auth().verifyIdToken()` requires Firebase Admin SDK to be initialized. Check `config/firebase-admin.js` exists and `GOOGLE_APPLICATION_CREDENTIALS` is set in `.env`
2. **Edge cases** — `email` may be `null` from Firebase if user hasn't verified email; app should handle empty email gracefully
3. **Token refresh** — Firebase ID tokens expire after 1 hour; app's `getIdToken()` auto-refreshes, so this is handled
